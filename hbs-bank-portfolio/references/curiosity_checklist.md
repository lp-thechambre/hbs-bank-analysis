# Curiosity Checklist v0.3

> 从 HBS 方法论 v0.3 蒸馏的 Portfolio 横评探测性问题。
> **核心原则：每道题都是排序、聚类或找异常值。不做单家银行的重新审计——那是 Depth 的工作。**

---

## 使用说明

AI 工作流：

```
1. 读全部 depth 摘要 + narrative，形成整体印象
2. Phase 1 宏观校准（Tier 1 必检）→ 决定整体攻防取向
3. Phase 2 排名与异常值扫描（Tier 2 触发式）→ 每道题产出一张排名或一张 2x2
4. 过程中补充自发问题
5. Phase 3 压测 + 健康度（Tier 1 必检，权重初稿后执行）
6. 基于全体排名共识 → 确定各家在 VOH 区间内的位置 → 输出战略权重
```

## Phase 1: 宏观校准（Tier 1 必检，3 条）

### 1.1 利率敏感度聚类

**问题**：按负债结构将银行分为三组——零售存款主导 / 对公存款主导 / 同业负债依赖。当前利率方向下，哪组最受益？哪组先受伤？

- 输出：一张 3 列分组表 + 每组合计市值权重
- 如果高权重银行集中在"利率不利"组 → 组合层面对利率方向有方向性暴露

**方法论**：Ch5 存款结构、Ch9 NIM

### 1.2 信用周期定位

**问题**：银行的不良/逾期比率是**普遍恶化**（系统性）还是**个别恶化**（特质性）？

- 输出：不良/逾期比率排列表 + 标注恶化是集中还是分散
- 如果是系统性恶化 → 整体降仓位或偏防御型银行
- 如果是个别恶化 → 那些恶化的银行是否恰是高权重？

**方法论**：Ch12 资产质量、Ch13 信用风险

### 1.3 公开叙事 vs 微观真相

**问题**：当前市场对银行业的共识叙事是什么？depth 微观数据聚合后，有没有与共识相悖的信号？

- 输出：共识叙事 x 1 + 矛盾信号 x N
- 例：共识说"NIM 全面承压"，但 5 家零售型银行存款成本在改善 → 市场可能过度悲观

**方法论**：Ch1 三源交叉验证

## Phase 2: 排名与异常值扫描（Tier 2 触发式，10 条）

每道题的核心动作：**把银行排成一列，看谁站两端、谁站错了位置。**

### 2.1 VOH vs 市值：谁被市场错判？

**问题**：将银行按 VOH 得分（纵轴）和市值权重（横轴）画一张 2x2。

```
         VOH 高
           │
  象限 II  │  象限 I
  (VOH高   │  (VOH高
   市值小) │   市值大)
           │
  ─────────┼────────── 市值
           │
  象限 III │  象限 IV
  (VOH低   │  (VOH低
   市值小) │   市值大)
           │
         VOH 低
```

- **象限 I**（VOH 高 + 市值大）：市场共识 + 分析共识一致，标配
- **象限 II**（VOH 高 + 市值小）：分析看好但市场忽略 → **潜在金子，重点调查**
- **象限 III**（VOH 低 + 市值小）：两方都不看好，低配
- **象限 IV**（VOH 低 + 市值大）：市场在扛但分析不看好 → **潜在地雷，重点调查**

**触发条件**：必检（这是后续排雷/找金子的入口）

**方法论**：Ch4 VOH

### 2.2 VOH vs Resilience：高分低韧的异常值

**问题**：将银行按 VOH 得分和 Depth 的 resilience 评分排成散点图。

- **异常值 A**：VOH 高 + resilience 低 → VOH 可能高估了管理层质量。查：VOH 三个子项中，是不是成长得分在撑，而分红和分散化得分平平？
- **异常值 B**：VOH 低 + resilience 高 → 市场和分析都低估了这家银行的管理层韧性

**触发条件**：存在明显偏离对角线的银行（AI 目测即可，不需统计算偏离度）

**方法论**：Ch4 VOH、Ch19 韧性

### 2.3 不良认定严格度排名

**问题**：银行按不良/逾期比率从低到高排列。

- 比率最低的三家和最高的三家分别是谁？
- 高权重银行集中在排名的哪一端？如果集中在低端（< 0.9）→ 组合在赌"认定宽松"这个因子
- 排名的中间值（median）在哪？与行业均值对比？

**触发条件**：高权重银行 >= 2 家在比率最低的三家中

**方法论**：Ch12.1、12.2

### 2.4 分红纪律排名

**问题**：按三个维度交叉排名：
- CDP 从低到高
- 冲击年份分红行为（降/冻/加）
- 分红稳定性（每股分红连续增长的年份数）

**综合排名垫底的银行**：CDP 高 + 冲击年份加息 + 分红不稳定 → 管理层可能短视
**综合排名顶端的银行**：CDP 低 + 冲击年份降息 + 分红稳定增长 → 管理层审慎且有长期视野

**触发条件**：存在 CDP > 40% 或冲击年份加息的银行（大概率会有）

**方法论**：Ch3.4、3.5

### 2.5 资本缓冲排名

**问题**：银行按 CET1 超出监管要求的缓冲空间（CET1 - 监管底线）从低到高排列。

- 缓冲最薄的三家合计在组合中权重多少？
- 缓冲最薄的三家，是否也是 CDP 最高或分红最激进的那三家？（交叉验证）

**触发条件**：存在 CET1 缓冲 < 2% 的银行

**方法论**：Ch15.3.1、15.3.5

### 2.6 行业敞口聚类

**问题**：将银行按最大行业敞口分组。如果三家以上高权重银行挤在同一组 → 组合在该行业上是 naked long。

- 不只要看行业名称，还要看关联行业（房地产-建筑-建材、城投-基建）
- 各组合计权重是多少？

**触发条件**：>= 3 家银行最大行业敞口相同或关联

**方法论**：Ch13.4 HHI、Ch18 房地产

### 2.7 边际效率趋势排名

**问题**：银行按**边际利息产出**（Δ利息收入/Δ利息支出）的三年趋势排序。

- 持续下降的 3 家：在以价换量，规模扩张没有效率支撑
- 持续改善或稳定的 3 家：规模扩张仍有纪律
- VOH top 3 的银行在效率排名中处于什么位置？

**触发条件**：必检（效率趋势是 VOH 内部一致性的验证）

**方法论**：Ch9.5

### 2.8 Integrity 排名 vs 风险暴露

**问题**：银行按 integrity 得分从低到高排列。垫底的三家：

- 在组合中合计权重多少？
- 在行业敞口聚类中是否恰好也在风险敞口最大的组里？（低 integrity + 高集中度 = 高危组合）
- 与拨备覆盖率 CV 最大的三家是否重叠？（财技使用的多维度验证）

**触发条件**：存在 integrity < 75 的银行

**方法论**：Ch20 诚信审计

### 2.9 房地产转型进度排名

**问题**：银行按房地产敞口下降幅度排序。

- 排在前面的：敞口实质性下降 + 新方向（绿色/科技/普惠）贷款增速和质量双升
- 排在后面的：声称转型但敞口不降 + 关联行业敞口也不降
- 高权重银行集中在排名的哪一端？

**触发条件**：>= 3 家银行房地产敞口 > 25%

**方法论**：Ch18

### 2.10 行业预见力验证

**问题**：如果宏观判断中识别出近年的特定行业风险事件（如 2021-2024 房地产暴露），将银行分为三组：

- **先行者**：在风险暴露前 1-2 年已主动降低该行业敞口
- **跟随者**：风险暴露当年开始调整
- **落后者**：至今敞口仍高或被迫大幅核销

三组各自在组合中的权重分布？

**触发条件**：宏观环境中存在可识别的行业风险事件

**方法论**：Ch13.5 后验验证框架

## Phase 3: 压力测试与健康度（Tier 1 必检，4 条）

战略权重初稿形成后执行。

### 3.1 利率冲击传导

**问题**：如果利率 +100bp，按当前权重排序，哪家银行先崩、哪家几乎不受影响？

- 输出：一张从"最脆弱"到"最坚韧"的排序表 + 脆弱银行的合计权重
- 判断依据：同业负债依赖度、资产久期、depth 中的 NIM 敏感度标注

**方法论**：Ch5、Ch9、Ch14

### 3.2 信用冲击缓冲

**问题**：如果不良率普遍 +200bp，拨备覆盖率最低的三家能扛多久？这三家合计权重是否可接受？

**方法论**：Ch12、Ch15

### 3.3 隐性同源风险

**问题**：扫描全部银行的 narrative 和敞口数据，是否存在一个风险因子是多达 5 家以上银行同时在赌的？

- 可能的因子：利率下行、长三角经济、零售转型、房地产回暖
- 不论权重如何，先识别出来，再判断暴露程度

**方法论**：Ch13.4 HHI 精神、Ch17 地区

### 3.4 业务与地域覆盖

**问题**：高权重 top 5 在业务类型（零售/对公/综合）和区域（长三角/珠三角/环渤海/中西部/全国性）上是否过度集中？

- 如果全是一种类型 → 该类型对宏观变量的敏感度即组合的敏感度

**方法论**：Ch1 银行类型判定、Ch17

## 附录：权重框架

Checklist 产出排名共识。权重由一条方程计算：

```
w_i = mcap_i + (市值排名_i - VOH(portfolio)排名_i) x sigma_mcap

sigma_mcap = 银行市值权重的标准差
```

Checklist 的作用：调整 VOH(depth) 排名 -> VOH(portfolio) 排名。调整的是排序，不调分数，附调整理由。

后处理：负数 clip 到 0 -> STRONG_SELL = 0 -> SELL <= 3% -> 单只上限裁剪 -> 归一化。
