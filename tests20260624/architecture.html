HBS-Screen Pipeline Architecture

42 A-share listed banks → 10–15 screened candidates  |  ■ Python script   ■ AI spawn   ■ Disk artifact

Python Layer 0 — Data Engineering 全脚本，无 AI 参与
fetch_financials.py
从东方财富 F10 API 拉取 42 家银行的主要财务指标、利润表、股息数据
产出 raw_main.json raw_profit.json raw_dividends.json
pb_fetcher.py
从腾讯财经 qt.gtimg.cn 批量拉取股价，用于 PB / EPS yield 计算
产出 raw_prices.json
env_scan.py
运行前环境检查：Python 版本、依赖包可用性
产出 env_scan.json
↓
generate_bank_cards.py
读入 raw JSON → 银行分类 (利息收入占比) → 同行统计 → 五维度评分 → 生成卡片和 index.csv。

评分方式: D1/D3/D4 同行百分位 + D2 绝对线性 (pcr/3.0, loan_prov*20) + D5 piecewise 阈值
产出 index.csv cards/*.md ×42 generation_metadata.json
generate_embeddings.py
调用 OpenAI 兼容 embedding API 对 42 份卡片生成向量 → 纯 Python KMeans 聚类 → 轮廓系数评估。
API 不可用时降级为模拟 embedding。
产出 cluster_report.json
Layer 0 磁盘产出: index.csv (42行)  |  cards/*.md (42份, ~1000-1500 tokens/份)  |  cluster_report.json  |  generation_metadata.json  |  env_scan.json
⬇
AI spawn ×2 Layer 1 — Quantitative + Edge Signal Detection 并行运行，互不可见
quant_spawn (AI)
角色: 量化分析师。读取 index.csv + cards/*.md，按阈值规则 (CET1<8.5%→REJECT, NPL>3%→REJECT, ROE<5%→WATCH...) 对 42 家银行做定量初判。
prompt: quant_spawn_prompt.md (~94行)
防幻觉: "Never make up data. Every value must come from index.csv or a card."
限制: 只能读 15-20 张卡片，不引用 flag ID (F1-F11)，不接触 edge 结果。
产出 quant_markers.json   输入 index.csv cards/*.md
edge_spawn (AI)
角色: 异常检测器。读取 cards/*.md，找五类异常信号：metric_extreme, metric_mismatch, group_outlier, trend_break, external_signal。
prompt: edge_spawn_prompt.md
防幻觉: "Normal data missing (NA) is NOT an anomaly." external_signal 必须引用搜索结果和日期，不得用"未经证实的传闻"。
限制: 最多 3 次 web_search，不接触 quant 结果。
产出 edge_markers.json   输入 cards/*.md
Layer 1 磁盘产出: quant_markers.json (42 家银行的 PASS/WATCH/REJECT + curiosity 字符串)  |  edge_markers.json (异常银行标记 + 严重度)
⬇
Python Sidecar — Pipeline Statistics (确定性统计，不参与 AI 推理)
compute_pipeline_stats.py
读取上游 quant_markers + edge_markers + index.csv，做确定性计数和交叉验证： 统计各层通过/拒绝数、银行类型分布、排名声明真实性标注。
这是 synthesis 层的防幻觉锚点——所有聚合统计必须来自此文件，AI 禁止自行数数。
产出 pipeline_stats.json
⬇
AI spawn ×3~4 Layer 2 — Qualitative Deep Read (by Bank Type Group) 并行运行，每组 ≤10 家银行
qual_spawn — Group A (AI)
国有大行 (6家): 工商/建设/农业/中国/交通/邮储
焦点: NIM 压缩、股息可持续性、TLAC 达标
产出 qual_markers_group_a.json
qual_spawn — Group B (AI)
股份制 (9家): 招商/兴业/浦发/中信/民生/光大/平安/华夏/浙商
焦点: 零售转型、财富管理收入占比、房地产敞口
产出 qual_markers_group_b.json
qual_spawn — Group C (AI)
城商行 Tier 1 (≥1万亿总资产): 江苏/北京/上海/南京/宁波...
焦点: 区域集中度、对公贷款质量
产出 qual_markers_group_c.json
qual_spawn — Group D (AI)
城商行 Tier 2 + 农商行 (<1万亿)
焦点: 拨备充足性、规模瓶颈、并购风险
产出 qual_markers_group_d.json
输入: cards/*.md + quant_markers.json + edge_markers.json (必须回应上游 flag)  |  Prompt: qual_spawn_prompt.md  |  防幻觉: "Do NOT contradict upstream markers without card-level evidence. Rank claims MUST be scoped to your group only — NEVER '#1 in universe'."  |  输出: PASS / WATCH / REJECT + upstream_flags_responded_to
已知问题: qual_spawn_prompt 要求 upstream_flags_responded_to 引用 flag ID (如 "quant:F4")， 但 quant_spawn 输出格式只含 curiosity 字符串，没有 flag ID。flag ID (F1-F11) 定义在 scoring_rules.md 中， quant prompt 从未引用它们，导致跨层追溯链断裂。
⬇
AI spawn ×1 Layer 3 — Synthesis (Final Judge)
synthesis_spawn (AI) — "Final Judge"
角色: 读取所有上游 markers + pipeline_stats.json，将银行分为四类处理：
  A) 一致通过 → HIGH_CONFIDENCE_PASS → 直接入选
  B) 一致拒绝 → UNANIMOUS_REJECT → 直接淘汰
  C) 分数边缘 → 读卡核实
  D) 层间冲突 → 读卡仲裁

硬约束:   • 所有聚合计数必须来自 pipeline_stats.json，严禁 AI 自行数数   • 卡片读取预算 ≤ 5 张   • GREEN 银行必须验证 CET1 ≥ 9.5%   • 每家银行必须有且仅有一个 tier 和一个 selection_path   • 拒绝银行必须有结构化条目 (rule/metric/value/threshold)
Prompt: synthesis_spawn_prompt.md (~390行，防幻觉最强)
产出 final_output.json screening_report.md 输入 quant_markers.json edge_markers.json qual_markers_*.json pipeline_stats.json index.csv cluster_report.json
最终产出: final_output.json (10-15 candidates + rejected + analyst_notes)  |  screening_report.md (可读 Markdown)

compute_scores.py — 独立评分管线 (非 AI)

这条管线完全不经过 AI，直接对原始数据进行确定性评分。与 AI spawn 管线 没有数据流对接。

Python compute_scores.py — 五阶段评分引擎
phase1
粗筛: R1(CET1<8.5%) → R2(NPL>3%) → R3(PCR<120%) → R4(净利润为负) → R5(关键字段缺失>3)
最先命中的拒绝规则生效 (continue 跳过后续)
产出 phase1_results.json (42→~28)
phase2
五维度评分: D1-D5 全部同行百分位 (含 D2)。
D3 RORWA 用硬编码阈值估计 (0.5%/0.8%/1.0%/1.5%/2.0%)。
D4 用总资产排名作为增长的代理变量。
产出 phase2_results.json (含 composite score)
triggers
Phase 3 触发: A) 边缘分数 (±5% cutoff) B) ≥2 WATCH flag C) 高盈利+差资产质量冲突 D) REJECT flag 需确认
强制 5-10 家触发。
产出 phase3_triggers.json
finalize
编译最终输出: 按 composite score 排序 → 取 top 15 → 生成最终 JSON。
注意: phase1 的 rejected 列表 没有在 finalize 中使用。
产出 final_output.json
输入: raw_main.json + raw_profit.json + raw_dividends.json + raw_prices.json (与 L0 相同)  |  运行方式: python3 compute_scores.py --mode phase1 ... && --mode phase2 ... && --mode triggers ... && --mode finalize ...
与 AI 管线的评分差异 (D2 最关键):
compute_scores D2: 全部同行百分位 (_linear_score)
generate_bank_cards D2: pcr/3.0 + loan_prov*20 (绝对线性)
→ 同一家银行，两边的 D2 分数可能差 2-3 倍。卡片上的 D2 分数 ≠ 最终排名的 D2 分数。

关键架构问题

1. 双评分管线没有数据对接
AI 管线 (L0→L1→L2→L3) 和 compute_scores.py 各自独立产出 final_output.json。 AI 读的卡片分数 (D2=绝对值) 和 compute_scores 的排名分数 (D2=百分位) 互不可见。 最终的 final_output.json 到底用哪边的分数？目前没有明确仲裁。
2. Flag ID 追溯链断裂
scoring_rules.md 定义了 F1-F11 flags → quant_spawn 输出 curiosity 字符串 (无 flag ID) → qual_spawn 要求引用 "quant:F4" 格式 → flag ID 不存在于 quant 输出中。qual 层无法结构化关联上游标记。
3. 卡片标注不诚实
卡片 "Dimension Scores" 表头写 "Peer-Group Relative"，但 D2 子项 (PCR, LPR) 实际用的是绝对线性映射，不是同行百分位。AI 在 L1/L2 读卡时基于错误标签做判断。
4. 银行分组不同步
bank_constants.py、compute_scores.py、bank_list.md、qual_spawn_prompt.md 各有一份银行分组/类型覆盖。SH601825 (沪农商行) 在一处标 rural_commercial 另一处标 city_commercial，SH601077 (渝农商行) 分组存疑。
